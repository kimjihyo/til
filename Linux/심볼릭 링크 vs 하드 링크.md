
**링크**


리눅스에는 두가지 종류의 링크가 있다.


| **링크 종류**                                      | **설명**                    | **파일 시스템 수준** | **`ls -l`** **표시** | **대표 옵션** |
| ---------------------------------------------- | ------------------------- | ------------- | ------------------ | --------- |
| **하드 링크 (hard link)**                          | 실제 파일 데이터에 대한 **또 다른 이름** | inode 공유      | 일반 파일처럼 표시됨        | 기본 (`ln`) |
| **심볼릭 링크 (symbolic link)**= 소프트 링크 (soft link) | 실제 파일 경로를 가리키는 **얇은 포인터** | inode 다름      | `l`로 시작, → 화살표 표시  | `ln -s`   |


**ln 의 옵션**


| **옵션** | **설명**                    |
| ------ | ------------------------- |
| `-s`   | 심볼릭 링크 생성                 |
| `-f`   | 기존 파일 덮어쓰기                |
| `-n`   | 기존 심볼릭 링크 위에 덮어쓰기 허용      |
| `-T`   | 링크 이름이 디렉터리여도 강제로 이름으로 취급 |


**언제 하드링크를 쓰고 심볼릭 링크를 사용하나?**


| **목적/상황**                   | **추천**                   |
| --------------------------- | ------------------------ |
| 파일 복제처럼 보이되 용량은 아끼고 싶다      | ✅ **하드링크**               |
| 원본 파일이 삭제돼도 데이터가 남아있게 하고 싶다 | ✅ **하드링크**               |
| 다른 파티션/디렉터리 구조에서도 링크 만들고 싶다 | ❌ 하드링크 불가 → ✅ **심볼릭 링크** |
| 가시적으로 '바로가기' 형태를 만들고 싶다     | ✅ **심볼릭 링크**             |
| 디렉터리를 링크하고 싶다               | ✅ **심볼릭 링크**             |
